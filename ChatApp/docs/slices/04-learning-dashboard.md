# 切片：学习看板

## 目标

用户可以在学习 Tab 查看已加入课程、学习概览和后端记录的课程进度，并从课程卡片继续进入对应课程教材。

## 用户流程

1. 用户打开学习 Tab。
2. App 调用 `GET /api/frontend/dashboard`。
3. App 渲染已加入课程、教材数量和累计学习时长。
4. 如果没有已加入课程，用户可从空状态跳转到课程 Tab。
5. 如果已有加入课程，课程卡片展示进度、当前章节和“继续学习”入口。
6. 用户点击“继续学习”进入对应课程详情页。

## API

- `GET /api/frontend/dashboard`

## 页面

- `DashboardScreen`

## 组件

- `AppCard`
- `AppButton`
- `StateView`

## 状态

- loading
- empty
- error
- normal

## 不做范围

- 课程详情
- 教材列表和阅读
- 章节级精确定位
- 在看板内直接打开具体章节
- AI 问答
- 教材上传、提炼和向量化

## 验收标准

- 学习看板通过 `frontendService` 加载。
- 网络失败时显示可重试错误状态。
- 没有已加入课程时显示空状态，并可跳转到课程 Tab。
- 有已加入课程时显示课程数、教材数、学习时长、课程进度和课程卡片。
- “继续学习”进入对应课程详情页，不再是禁用占位。
- Screen 层没有直接 `fetch`。
